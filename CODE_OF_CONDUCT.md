# Code of Conduct

The Liquid project aims to be a welcoming, harassment-free environment
for everyone — regardless of experience level, background, or identity.

> **Status:** Liquid is **pre-alpha**, maintained in spare time by a
> single person. **No formal enforcement process, response-time
> commitment, or dedicated reporting alias applies until the project
> tags its first stable release (`v1.0.0`).** This file describes the
> standard the project intends to follow and how to report concerns
> in the meantime.

## We adopt the Contributor Covenant

Liquid follows the **Contributor Covenant, version 2.1**, the de-facto
industry-standard code of conduct used by Rust, Flutter, Node.js,
Kubernetes, and most major open-source projects.

The full text is published at:

> <https://www.contributor-covenant.org/version/2/1/code_of_conduct/>

We adopt that text by reference rather than copying it inline, so
updates to the standard reach this project without manual sync. The
canonical version always wins.

## Scope

This Code of Conduct applies in every project space — the GitHub
repository, issues, pull requests, discussions, code review comments,
and any project chat or event spaces if and when those exist.

## Reporting concerns (pre-1.0)

There is no dedicated reporting alias yet. Until the project tags
`v1.0.0`, please send concerns through one of:

- **GitHub's private vulnerability reporting** (re-purposed for serious
  CoC reports because it routes privately to repository
  administrators):
  <https://github.com/open-equilibrium/liquid/security/advisories>
- **Direct message** to one of the repository administrators on
  GitHub.

Reports are looked at as the maintainer's time allows. The maintainer
will treat them as confidential, but cannot commit to a specific
response window pre-1.0. Reports related to active project spaces
(issues, PRs, discussions) get priority over those about external
spaces.

## Enforcement actions (intended)

When responding to incidents, the maintainer intends to follow the
**Contributor Covenant Enforcement Ladder** for severity calibration:

> <https://www.contributor-covenant.org/version/2/1/code_of_conduct/#enforcement-guidelines>

The ladder ranges from a private written note, through temporary bans
from interaction, up to a permanent ban. Specific responses are
calibrated to severity and pattern of behaviour. Pre-1.0, enforcement
is best-effort and at the maintainer's discretion.

## What's planned at v1.0 (not in effect yet)

Tracked under the *Pre-1.0 obligations checklist* in
[`IMPLEMENTATION_PLAN.md`](IMPLEMENTATION_PLAN.md):

- A real `conduct@…` contact alias on a project-owned domain
- A documented response-time target (acknowledge within N business
  days, etc.)
- Whether enforcement is single-maintainer or delegated to a small
  CoC committee
- Public reporting of CoC actions in aggregate (counts, not names) on
  a regular cadence
